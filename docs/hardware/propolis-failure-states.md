# Propolis Failure States

Propolis is the failure, quarantine, and recovery system.

It turns mistakes into gameplay instead of pure punishment.

## What Causes Propolis

A Nectar Node can become Propolis from:

- overheating
- failed overclocking
- power surge
- bad cooling layout
- ignored maintenance
- event damage
- failed repair attempt

## Propolis State

When a node enters Propolis:

- hash output becomes 0%
- node remains visible in the rack
- rack slot is blocked
- player gets a repair/salvage choice
- Tribunal may inspect the node
- rare recovery outcomes become possible

## Player Choices

| Choice | Outcome |
|---|---|
| Repair | Restore the node for HoneyCoin/Wax cost |
| Salvage | Destroy node for Wax/Nectar Shards |
| Tribunal Inspect | Chance to recover rare variant |
| Quarantine | Store for later repair |
| Royal Jelly Restore | Premium late-game recovery path |

## Rare Recovery

Some nodes can become special variants after Propolis recovery.

Examples:

| Original Node | Rare Recovery |
|---|---|
| Vintage Nectar S9 | Scarred S9 |
| Bombus HeatCore | Charred HeatCore |
| Nectar Hydro | Frostbitten Hydro |
| Royal Hydro Node | Royal Scar Hydro |
| BlackComb Titan | BlackComb Relic |

## Design Purpose

Propolis creates:

- risk
- decision-making
- repair economy
- salvage economy
- rare collector variants
- stories players remember

## Player-Facing Message Examples

"Your Vintage Nectar S9 overheated and entered Propolis."

"Repair, salvage, or send to Tribunal?"

"The Tribunal recovered a Scarred S9 variant."

## Anti-Frustration Rule

Propolis should hurt, but it should not make players quit.

Starter nodes should have low repair costs.

Rare nodes should have meaningful recovery options.

Mythic nodes should never be permanently destroyed without clear player choice.
