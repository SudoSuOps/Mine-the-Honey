# Player Hosting Loop

Hosting lets players rent Hive rack space to other players.

It turns the Hive into a social datacenter.

## Core Idea

A player with strong power and cooling can host another player's Nectar Nodes.

The host earns fees.

The guest earns mining yield.

Both players benefit.

## Host Stats

| Stat | Meaning |
|---|---|
| Uptime | How reliable the host Hive is |
| Electric Rate | Fee charged for power |
| Cooling Grade | How safe the Hive is |
| Rack Slots | Number of available hosted slots |
| Tribunal Reputation | Trust score |
| Accepted Nodes | Which miners the host allows |

## Guest Benefits

Guests can:

- mine without owning a large Hive
- use better cooling
- reduce Propolis risk
- access better uptime
- participate in host events

## Host Benefits

Hosts earn:

- hosting fees
- electric rate margin
- Tribunal reputation
- social traffic
- leaderboard status

## Risks

Bad hosts can cause:

- overheating
- downtime
- reduced yield
- Propolis events

The Tribunal should track hosting reputation.

## Design Goal

Hosting makes advanced players feel like operators.

Newer players get access to better infrastructure.

The social economy becomes part of the game loop.
